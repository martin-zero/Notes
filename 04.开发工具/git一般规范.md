# Git 一般规范

## 1. 接手工程
刚接手工程首先要将工程代码从远端仓库克隆到本地
```sh
git clone 仓库地址
```

## 2. 开发新功能
每次需要开发一个新功能，一般需要创建一个新的分支进行功能开发，首先我们需要拉取主干分支为最新版本
```sh
git pull
```
之后我们基于该分支创建一个新的分支
```sh
git checkout -b 分支名称
```
创建分支之后我们就可以在对应分支开发对应的功能了

## 3. 提交
每当我们开发完成一个小功能点都要为其创建一个commit，一般规定每一个commit只实现一个单一的功能点。首先我们将需要提交的文件或者文件中的行添加到待提交区。
```sh
git add 文件名
```
当将待提交的文件都添加之后就可以编写commit了
```sh
git commit 
```
一般每天或者一定时间需要把本地提交同步给远端
```sh
git push
```

## 4. 保持最新
有时候主干分支会有新的提交或者bug修复，我们需要定期保证我们的分支为基于最新版本的主干分支代码。首先我们从云端更新主干分支
```sh
git fetch origin
```
之后将我们的功能分支变基到主分支
```sh
git rebase origin/主分支名称
```
变基之后我们想要push到远端需要使用强制推送
```sh
git push --force-with-lease
```

## 5. 合入主干分支
当该分支功能彻底开发完成后，我们需要将分支合入主干分支
```sh
git checkout 主分支名称
git merge 分支名称
git push
```

## 6. 查看git历史
```sh
git reflog
```
