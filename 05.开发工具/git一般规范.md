## 1.接手工程
刚接手工程首先要将工程代码从远端仓库克隆到本地
```sh
git clone 仓库地址
```

## 2.开发新功能
每次需要开发一个新功能，一般需要创建一个新的分支进行功能开发，首先我们需要拉取主干分支为最新版本
```sh
git pull
```
之后我们基于该分支创建一个新的分支
```sh
git checkout -b 分支名称
```
创建分支之后我们就可以在对应分支开发对应的功能了

## 3.提交
每当我们开发完成一个小功能点都要为其创建一个commit，一般规定每一个commit只实现一个单一的功能点。首先我们将需要提交的文件或者文件中的行添加到待提交区。
```sh
git add 文件名
```
当将待提交的文件都添加之后就可以编写commit了
```sh
git commit 
```
一般每天或者一定时间需要把本地提交同步给远端
```sh
git push
```
